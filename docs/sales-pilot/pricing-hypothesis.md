# Pricing Hypothesis

This is for internal validation only. Do not quote externally until pilots prove
repeatable value.

## Packaging Assumption

Start with a services-assisted pilot, then move toward repeatable productized
packages.

## Pilot Package

Two-week QA Memory Pilot:
- one target area;
- one generated QA memory pack;
- one validation report;
- one review session;
- one expansion recommendation.

Internal price proxy:
- 20 to 40 engineering hours equivalent;
- value measured against reduced QA context gathering and faster release
  readiness decisions.

External hypothesis:
- fixed pilot fee for first target area;
- follow-on monthly subscription or support retainer if teams reuse the pack.

## Expansion Tiers

1. Single workflow.
   One module, release area, or ETL pipeline.

2. Team memory.
   Multiple workflows, shared validation rules, and onboarding notes.

3. Controlled autonomy.
   Autopilot, QA review, business acceptance, and guardrail reporting with
   explicit human approvals.

## Value Metrics

- Hours saved gathering release context.
- Number of reusable test specs generated.
- Number of missing validation gaps found.
- Faster onboarding for QA engineers.
- Reduction in repeated QA handoff questions.

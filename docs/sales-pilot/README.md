# BQA-OS 2-Week QA Memory Pilot Package

This package supports internal validation of the BQA-OS pilot offer before any
external customer outreach.

Target buyer:
- QA Lead
- QA Automation Lead
- Engineering Manager responsible for test reliability
- Data-heavy B2B SaaS, fintech, and platform teams

Pilot promise:
Turn existing QA run evidence into a reusable, local-first QA memory pack:
agents, workflows, test specs, validation checklists, and next-step reports.

Contents:
- `one-pager.md` - pilot offer and scope.
- `demo-script.md` - 20 minute internal demo flow.
- `discovery-call-script.md` - qualification questions and success criteria.
- `onboarding-checklist.md` - required inputs and safety checks.
- `outreach-snippets.md` - Slack, LinkedIn, and email drafts.
- `pricing-hypothesis.md` - internal pricing and packaging assumptions.
- `stakeholder-faq.md` - answers for QA, engineering, security, and leadership.

Safety:
- Use synthetic or public-safe examples only.
- Do not paste private customer data, production logs, credentials, or raw
  session archives into committed files.
- Keep all real pilot artifacts local under `.bqa/` or `.bqa-team/`.
